# REST API reference

The API runs on port 8000 by default. OpenAPI / Swagger UI is served
live at [`/docs`](http://localhost:8000/docs) and ReDoc at
[`/redoc`](http://localhost:8000/redoc). This page summarises the
most-used endpoints.

## Authentication

Most endpoints accept one of two auth styles:

* **Agent HMAC** — for SDK / agent traffic. `Authorization: Bearer
  <api_key>` plus `X-Signature: <hmac-sha256>`.
* **Workspace JWT** — for dashboard traffic. `Authorization: Bearer
  <auth0_access_token>`. When the local identity provider is active,
  `X-User-Sub: <user_id>` is accepted instead.

## Core

| Method | Path                       | Notes                                       |
|--------|----------------------------|---------------------------------------------|
| POST   | `/api/v1/request`          | Submit a new approval request.              |
| GET    | `/api/v1/status/:job_id`   | Poll for terminal state.                    |
| POST   | `/api/v1/test-request`     | Dry-run a request (no execution).           |
| POST   | `/jobs/:id/decision`       | Submit an approve/deny from the dashboard.  |

## Rules

| Method | Path                  | Notes                                  |
|--------|-----------------------|----------------------------------------|
| GET    | `/rules`              | List rules in the active workspace.    |
| POST   | `/rules`              | Create a rule.                         |
| PATCH  | `/rules/:id`          | Update a rule.                         |
| DELETE | `/rules/:id`          | Delete a rule.                         |
| POST   | `/rules/simulate`     | Dry-run a request against all rules.   |

## Approvers, Agents, Connections

| Method | Path                  | Notes                                  |
|--------|-----------------------|----------------------------------------|
| CRUD   | `/approvers`          | Workspace-scoped approver registry.    |
| CRUD   | `/agents`             | Per-agent API keys, HMAC, revocation.  |
| CRUD   | `/connections`        | Connected services + OAuth bootstrap.  |

## Local approvals (only when `APPROVAL_PROVIDER=local`)

| Method | Path                                    | Notes                       |
|--------|-----------------------------------------|-----------------------------|
| GET    | `/local-approvals/:handle`              | Inspect a pending request.  |
| POST   | `/local-approvals/:handle/approve`      | Mark approved.              |
| POST   | `/local-approvals/:handle/reject`       | Mark rejected.              |

## Health

| Method | Path             | Notes                                          |
|--------|------------------|------------------------------------------------|
| GET    | `/health`        | Liveness + shallow DB ping.                    |
| GET    | `/health/deep`   | DB + Redis + Ollama connectivity.              |

For the complete, autogenerated list of endpoints and payload schemas,
visit `/docs` on a running instance.
